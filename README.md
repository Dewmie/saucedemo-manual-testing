# Sauce Demo Manual Testing Project

## Project Overview

This is a manual testing project for the Sauce Demo web application. Sauce Demo is a sample e-commerce website used for practicing software testing. The application includes login, product listing, cart, checkout, order completion, menu options, and different user-specific behaviors.

This project was created to demonstrate manual QA skills, test case writing, test execution, bug reporting, and test summary preparation.

## Application Under Test

- Application Name: Sauce Demo
- Website: https://www.saucedemo.com/
- Application Type: Web-based e-commerce demo application
- Testing Type: Manual Testing

## 📋 What's Inside the Excel File In test-documents Folder

### Sheet 1 — Test Plan
Scope, entry/exit criteria and other relavant details.

### Sheet 2 — Test Cases (42 test cases)

The test cases were prepared in Excel using the following fields:

| Field | Description |
|---|---|
| Test case ID | Unique ID for each test case |
| Test Case | Short description of the test |
| Pre Conditions | Conditions needed before executing the test |
| Test Data | Data used during execution |
| Test Steps | Step-by-step actions followed by the tester |
| Expected Results | Expected behavior of the application |
| Actual Result | What actually happened |
| Status | Pass or Fail result |

### Sheet 3 — Bug Report
9 bugs documented with severity, steps to reproduce and expected vs actual results.

### Sheet 4 — Test Summary
Overall pass/fail results.
